# 依存ライブラリ一覧

本ドキュメントはルール18に基づき、scliが使用するすべてのサードパーティ依存ライブラリを記録します。

---

## 直接依存

### `github.com/spf13/cobra`

| 項目 | 内容 |
|------|------|
| バージョン | v1.10.2 |
| ライセンス | Apache-2.0 |
| 目的 | CLIフレームワーク — コマンド/サブコマンドのディスパッチ、フラグ解析、ヘルプ生成 |
| 内製しない理由 | GoのCLI開発における事実上の標準ライブラリ。サブコマンドルーティング、フラグ継承、ヘルプ生成、シェル補完を再実装することは、メリットのない大きな工数となる。 |

---

## 間接依存

直接依存から引き込まれるライブラリ。scliのソースコードから直接インポートはしない。

### `github.com/fatih/color`

| 項目 | 内容 |
|------|------|
| バージョン | v1.19.0 |
| ライセンス | MIT |
| 目的 | ANSIカラー出力、全プラットフォームでのTTY自動検出 |
| 内製しない理由 | WindowsのANSIエミュレーションとisattyチェックをプラットフォーム横断で処理する。これを内製するとOSごとのコードが約200行増える。 |

### `github.com/zalando/go-keyring`

| 項目 | 内容 |
|------|------|
| バージョン | v0.2.6 |
| ライセンス | MIT |
| 目的 | クロスプラットフォームのOSキーチェーンアクセス（macOS Keychain / Linux libsecret / Windows Credential Manager） |
| 内製しない理由 | プラットフォームごとに異なるネイティブAPI（Security.framework、dbus/libsecret、wincred）が必要。内製するとこのライブラリをそのまま複製することになる。 |

### `github.com/joho/godotenv`

| 項目 | 内容 |
|------|------|
| バージョン | v1.5.1 |
| ライセンス | MIT |
| 目的 | 設定レイヤーのトークンフォールバックに使用する `.env` ファイルの解析 |
| 内製しない理由 | `.env` ファイルの解析にはクォートやコメントなど多くのエッジケースがある。セキュリティ上重要な処理に実績あるライブラリを使うことで微妙なバグを防ぐ。 |

### `github.com/danieljoos/wincred`
